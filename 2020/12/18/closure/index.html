闭包 – 本质 是一个函数内部创建另一个函数

  • 特性：

    1. 函数嵌套函数
    2. 函数内部可以引用外部参数和变量
    3. 参数和变量不会被垃圾回收机制回收

  • 结构：

    1. 函数作返回值

    1. 闭包作为参数传递

      优点:

      1. 保护函数内的变量安全，实现封装，防止变量流入其他环境发生命名冲突
      2. 在内存中维持一个变量，可做缓存
      3. 匿名自治性函数可减少内存消耗

      缺点：

      1. 被引用的私有变量不能被销毁，增大内存消耗，造成内存泄漏，解决方法：使用完变量后手动赋值为null
      2. 由于closure设计跨域访问，会导致性能损失。解决方法：把跨作用域变量存储在局部变量中，然后直接访问局部变量，减轻对执行速度的影响。

function 立即执行 （IIFE）

  • function fnname(){}
    • function关键字声明一个函数 + 函数名 => 函数声明
  • var fnname = function(){}
    • function关键字声明一个函数，无函数命名，是匿名函数——-（匿名函数属于函数表达式）
    • 把匿名函数赋予一个变量fnname => 函数表达式

解释：

  • IIFE形成的逻辑

    function a(){} —-函数声明

    a() —-函数调用

    function a(){}() —-立即执行 =>==混淆了函数声明和函数调用，产生歧义==

    解释：

    • 函数声明 与 函数调用 不能同时使用,括号使得函数声明 转化为=> 函数表达式，故解析器会将其作为函数表达式处理，只有程序执行到函数a时才会被访问（到了就执行）。
    • ==任何消除函数声明和函数表达式间歧义的方法，都可以被解析器正确识别。==
    • 用各种操作符告诉解析器（包括括号），这不是函数声明，而是函数表达式。其中一元运算符是消除歧义最快的方法。
    • =>一元运算符使语句表达式化 + 括号表示函数调用
    • =>即：==一元运算符使歧义可被识别，直接执行==

closure(闭包)

概念：内部函数总是可以访问其所在的外部函数中声明的参数和变量，即使在其外部函数被返回（寿命终结，return）了之后

另解：闭包找到的是用一地址中，父级函数中，对应变量最终的值

闭包函数：声明在一个==函数中的函数==

特点：

  1. 让外部访问函数内部变量成为可能

  2. 局部变量常驻在内存中

    • 每次外部函数执行的时候，外部函数的引用地址不同，都会重新创建一个新的地址(指引用该函数的引用地址)
    • 凡是当前活动对象中有被内部子集引用的数据，这个数据不会被删除，保留一个指针给内部活动对象(只要内部函数引用过外部函数的参数，就会保留该数据，即使外部函数寿命已经终结)

  3. 避免使用全局变量，防止全局变量污染

  4. 会造成内存泄漏———–(缺点)

闭包内存泄漏为：key = value，key被删除后，value常驻内存中，局部变量闭包升级版 => 自由变量 ？？

  1. 函数A中return函数B
  2. 变量C = 函数A的调用
  3. 调用函数C
